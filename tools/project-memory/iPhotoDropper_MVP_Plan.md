# План разработки iPhotoDropper (MVP)

## 1) Цель
Сделать нативное Windows-десктопное приложение для быстрого и надежного копирования фото/видео с iPhone по USB на ПК без «сбоев» и ручных танцев.

- Работа только с локальным USB-соединением (без Wi‑Fi).
- Упор на стабильность: контроль ошибок, повтор, атомарная запись, отчеты.
- Быстрый и простой UI для первого релиза.

## 2) Базовый стек и принципы
- **Платформа:** `C# + .NET 8`
- **UI:** `WinUI 3` (Windows App SDK)
- **Архитектура:** `MVVM` + DI (`Microsoft.Extensions.DependencyInjection`)
- **Логирование:** `Serilog` или `ILogger`
- **Асинхронная работа:** `async/await`, `CancellationToken`, `IProgress<T>`
- **Паттерны:** `Background Task Queue`, `Retry`, `Repository/Service` для доменных сервисов

## 3) Минимальный MVP (безопасный и «без проблем»)
1. Подключение iPhone по USB, определение устройства.
2. Проверка trust/доступности.
3. Скан фото/видео из камеры.
4. Выбор папки назначения.
5. Копирование в выбранную папку с прогрессом.
6. Лог ошибок и отчет по итогам импорта.
7. Защита от дублей и частичных файлов.
8. Пауза/возобновление/отмена.

## 4) Предлагаемая структура приложения
- **Core**
  - `iPhotoDropper.Core`
  - Сущности: `MediaItem`, `TransferJob`, `TransferResult`, `DeviceInfo`
  - Контракты интерфейсов: `IUsbDeviceService`, `IPhotoLibraryService`, `ITransferService`, `IHashService`
- **Infrastructure**
  - `iPhotoDropper.Infrastructure`
  - Реализация доступа к iPhone (через библиотеку/обертку под iOS USB)
  - Работа с файловой системой, проверка диска, очереди, повтор
- **Desktop/UI**
  - `iPhotoDropper.App`
  - WinUI 3 приложения: экраны, VM, компоненты прогресса
- **Data**
  - История/настройки (JSON или SQLite)
  - Логи и список ошибок импорта

## 5) Архитектурные сервисы
### Device Service (`IUsbDeviceService`)
- Обнаружение подключения/отключения устройств.
- События `DeviceConnected`, `DeviceDisconnected`.
- Проверка совместимости и состояния trust.

### Media Service (`IPhotoLibraryService`)
- Чтение списка медиа с атрибутами:
  - тип (`photo/video`)
  - уникальный ID/uuid
  - размер, MIME, путь на устройстве
  - дата съемки/создания
  - ориентация
- Получение превью (по возможности).

### Import Service (`ITransferService`)
- Очередь задач импорта (`single-thread` стартово, потом 2 max)
- Ограничение по скорости/параллельности (чтобы не перегружать USB)
- `Pause/Resume/Cancel`.
- Retry-policy: 2–3 попытки с backoff (например 1s → 2s → 4s).

### Duplicate/Integrity Service (`IHashService`)
- Пропуск дублей по ключу:
  - `deviceId + mediaId`
  - fallback: `(имя + размер + дата создания)`
- Запись в `*.part`, затем переименование после полного копирования.
- Проверка/лог успешной загрузки.

### Reporting Service
- Итоговый отчёт:
  - сколько найдено
  - сколько скопировано
  - сколько пропущено (дубль/ошибка)
  - путь назначения, время, ошибка по каждому неуспешному файлу

## 6) Поток работы пользователя
1. Подключить iPhone по USB.
2. Запустить приложение.
3. Выбрать устройство (или авто-выбор, если одно).
4. Нажать `Сканировать`. Приложение показывает список/счетчик.
5. Выбрать папку назначения + опции:
   - `Копировать только новые`
   - `Копировать фото + видео`
   - `Ограничить размер`
   - `Создать структуру по датам`
6. Нажать `Импортировать`.
7. Наблюдать прогресс и завершить.

## 7) Надежность и устранение сбоев (критично)
- `Atomic Write`: файл -> `filename.tmp` -> `filename`
- Проверка свободного места до запуска и перед каждой партии.
- Автоматическая пауза на 1 разрыв подключения + информативная ошибка.
- Хранить состояние незавершённых задач (для продолжения).
- Детальные коды ошибок:
  - `NoDevice`
  - `DeviceNotTrusted`
  - `TransferInterrupted`
  - `DiskFull`
  - `PermissionDenied`
  - `UnsupportedFormat`

## 8) UI план (WinUI 3)
- **Главный экран**
  - Статус устройства
  - Кнопки: `Поиск`, `Сканировать`, `Импорт`, `Пауза`, `Отмена`
  - Общий прогресс + прогресс по файлу
- **Панель настроек**
  - Путь назначения, фильтры, лимиты, формат структуры папок
- **Логи/история**
  - Список ошибок, кнопка открыть файл лога
- **Журнал завершения**
  - Чек-лист результатов копирования

## 9) План разработки (быстрый MVP)
- **День 1–2:** проект, скелет WinUI, DI, базовый экран, логирование.
- **День 3–4:** интеграция USB-сервиса/детекция устройства.
- **День 5–7:** получение списка медиа и базовый импорт.
- **День 8–10:** retry, дубли, atomic write, отчеты, ошибки.
- **День 11–12:** UI полировки, тесты на 2–3 устройствах, installer/MSIX.
- **День 13–14:** документация и релиз кандидата.

## 10) Риск-менеджмент
- **Риск:** нестабильность библиотеки доступа к iPhone на Windows.
  - **Митигировать:** выбрать наиболее стабильный путь подключения и держать fallback/обертку.
- **Риск:** зависания на больших файлах.
  - **Митигировать:** фоновые операции + прогресс + лимит одновременных копирований.
- **Риск:** проблемы доверия/безопасности.
  - **Митигировать:** понятные сообщения и понятная процедура trust на iPhone.

## 11) Что нужно подготовить сразу
- Visual Studio 2022 с WinUI workload.
- .NET 8 SDK.
- Apple Mobile Device/USB драйверы Windows.
- 2–3 тестовых iPhone разных версий iOS.
- Тестовая папка на ПК с достаточным свободным местом.

## 12) Что можно добавить после MVP (v1.1+)
- Выбор альбомов по датам/названиям.
- Экспорт настроек.
- Автоматический планировщик/фоновый мониторинг.
- Поддержка нескольких iPhone за раз (очередью).
- Поддержка RAW/HEIC профилей по желанию.
- Автосинхронизация и профили.
